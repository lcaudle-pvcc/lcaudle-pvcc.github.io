Chapter 6: Programming Languages
TERM DEFINITION
Machine language The language understood by a computer
Debugging The process of finding and fixing errors or bugs in the source code of any software
Assembler A program that takes basic computer instructions and converts them into a pattern of bits that the computer's processor can use to perform its basic operations.
Assembly language A type of low-level programming language that is intended to communicate directly with a computer's hardware.
Translator A program that converts source code into object code.
Compiler A special program that translates a programming language's source code into machine code, bytecode, or another programming language.
Interpreter Translates source code into object code one instruction at a time.
Cross-platform software Type of application/program/software that works on various operation system or devices, which are often called platforms.
Natural language A language that develops natually over time.
Formal language A language that is developed for a specific purpose.
Machine independence A term applied to software that is not dependent on the propertires of a particular machine, and can therefore be used on any machine.
High-level programming language Any programming language that enables development of a program in a much more user-friendly programming context and is generally independent of the computer's hardware architecture.
Object-oriented programming (OOP) A style of programming characterized by the identification of classes of objects closely linked with the menthods (or functions) with which they are associated.
Scripting language A programming langiage that employs a high-level construct to interpret and execute one command at a time.
Data types:
    Integer - Numeric data stored as whole numbers, can be positive or negative.
    Float - Numeric data stored using floating-point notation.
    String - Text data stored as characters.
Primitive data type Data type that is built into a programming language.
Data structure A specialized format for organizing, processing, retrieving and storing data.
Python collection data structures:
    List - A list is an ordered, and changeable, collection
    Tuple - A tuple is an ordered, and unchangeable, collection
    Set - A set is an unordered, and unchangeable, collection
    Dictionary - A dictionary is an unordered, and changeable, collection
Function A set of instructions used to perform a task.
Local variable A variable declared within a function that can only be used in that function.
Global variable A variable declared outside a function that can be used by any function.
Source program A program's original form
Object program A program's translation
