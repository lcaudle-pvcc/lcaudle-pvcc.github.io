All About GitHub

  1. Q. What is a version control system?
    A. A version control system is the practice of trackign and managing changes to sofware cofe. Allows software teams to make changes to codes in a more efficient way.
  2. Q. What is Git?
    A. Git is a free and open-source version control system that is designed to handle small and large projects.
  3. Q. What is GitHub?
    A. GitHub is a hosting service for software development and version control using Git.
  4. Q. What is the difference between Git and GitHub?
    A. Git is a version control system that allows you to manage and keep track of your source code history whereas GitHub is a cloud-based hosting service that allows you to manage your Git repositories.
  5. Q. Who started GitHub and how was it started?
    A. There are three main people who started GitHub, that being Tom Preston-Werner, Chris Wanstrath, and PJ Hyett, but Scott Chacon is also mentioned. They started it for their own work, and it was developed out of necessity.
  6. Q. What company owns it now?
    A. Microsoft Coportation
  7. Q. How much does a GitHub account cost?
    A. For basic individuals and organizations you can get it for free, for a team you can get it for $44 per user/year for the first year, after that it is $48/user/year, or for an enterprise it is $231 per use/year for the first year, and after that it is $252/user/year.
  8. Q. What is the Octocat?
    A. The Octocat is part cat, part octopus, and it is the mascot of GitHub.
  • Repository - A repository is the most basic element of GitHub. Imagine it as a project's folder, it contains all project files and stores any revision history.
  • Commit - Also a revision, it is an individual change to a file.
  • Fork - A personal copy of another user's repository that lives on your account. A fork allows you to make changes to a project without affecting the original repository.
  • Push - A push is sending your committed changes to a remote repository on GitHub.
  • Pull requests - Proposed changes to a repository submitted by a user and accepted or rejected by a repository's collaborators.
  • Workflows - A configurable automated process that will run one or more jobs.
  • Issues - Suggested improvements, tasks or questions related to the repository.
  • Raw button - Opens the file in raw form, meaning that any HTML formatting disappears.
  • Blame button - The blame feature in Git describes the last modification to each line of a fine, which generally displays the revision, author and time.
